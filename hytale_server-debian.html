01

Arquitectura del sistema

Así queda el sistema completo una vez montado. La PC servidor vive en una ubicación con buena conexión, enchufada permanentemente. El administrador gestiona todo de forma remota. Los jugadores se conectan a través de la red virtual de ZeroTier desde cualquier lugar.

Red ZeroTier Virtual
PC Debian Servidor · Ubicación remota
←── SSH + RustDesk ──→ administración remota
Administrador Gestión y mantenimiento
Hytale Server corriendo en segundo plano
Jugador A ZeroTier · remoto
Jugador B ZeroTier · remoto
💡

Los jugadores que estén físicamente en la misma red que el servidor pueden conectarse directamente por LAN sin usar ZeroTier, liberando esos slots de la red virtual para los que están en ubicaciones remotas.

02

Instalar Debian 12 Netinst

Descarga la imagen netinst (no la completa) desde debian.org/distrib/netinst. Pesa ~400MB y te deja instalar solo lo que necesitas.

⚠️

La netinst necesita internet durante la instalación para bajar paquetes. Conecta la PC por cable ethernet al router, no por WiFi, para minimizar pérdida de ancho de banda y evitar cortes durante la instalación.

// Durante el instalador

  1. ⚠️ Se recomienda conocimiento previo instalando Debian ⚠️
  2. Idioma, zona horaria, teclado → a tu gusto
  3. Particionado → Guided - use entire disk (simple y suficiente)
  4. Cuando pregunte el mirror de Debian → elige deb.debian.org
  5. En la pantalla Software selection → desmarca TODO excepto "standard system utilities". Nada de entornos gráficos aquí.
  6. Instala GRUB en el disco principal cuando lo pida
  7. Reinicia → arranca en terminal (TTY). Perfecto.
03

Post-instalación del sistema

Lo primero es corregir el sources.list y actualizar. Aquí está el contenido exacto para Debian 12:

// /etc/apt/sources.list

sources.list 
deb http://deb.debian.org/debian bookworm main contrib non-free non-free-firmware
deb http://security.debian.org/debian-security bookworm-security main contrib non-free non-free-firmware
deb http://deb.debian.org/debian bookworm-updates main contrib non-free non-free-firmware
💡

non-free-firmware al final de cada línea cubre drivers propietarios que no están disponibles en los repositorios oficiales de Debian por defecto. Inclúyelo aunque no creas necesitarlo, evita sorpresas con hardware como ciertas tarjetas WiFi.

// Actualizar e instalar esenciales

bash 
# Actualizar sistema completo
sudo apt update && sudo apt upgrade -y

# Instalar herramientas básicas
sudo apt install -y curl wget gnupg apt-transport-https \
  screen nano ufw openssh-server

// Instalar XFCE mínimo

bash 
sudo apt install -y xfce4 xfce4-terminal thunar

# Que el sistema arranque en modo texto por defecto (sin gráficos)
sudo systemctl set-default multi-user.target

Cuando quieras entrar al escritorio gráfico manualmente, desde la terminal escribe:

bash 
startx   # abre XFCE
# Para volver al modo texto: cierra sesión desde XFCE normalmente
04

Java — Adoptium Temurin 21

Debian netinst no trae Java. Hay que agregar el repositorio de Adoptium manualmente ya que no está en los repos oficiales de Debian.

bash 
# 1. Dependencias necesarias
sudo apt install -y wget apt-transport-https gnupg

# 2. Agregar la clave GPG de Adoptium
wget -qO - https://packages.adoptium.net/artifactory/api/gpg/key/public \
  | sudo gpg --dearmor -o /etc/apt/trusted.gpg.d/adoptium.gpg

# 3. Agregar el repositorio
echo "deb https://packages.adoptium.net/artifactory/deb bookworm main" \
  | sudo tee /etc/apt/sources.list.d/adoptium.list

# 4. Actualizar e instalar Temurin 21
sudo apt update
sudo apt install -y temurin-21-jdk

# 5. Verificar
java -version
✅

La verificación debe responder algo como OpenJDK Runtime Environment Temurin-21.x.x. Si ves eso, está perfecto.

05

ZeroTier — Red LAN virtual

ZeroTier reemplaza a Radmin VPN (que solo es Windows). Crea una red virtual privada donde todos los dispositivos se ven como si estuvieran en la misma LAN. Completamente gratis hasta 25 dispositivos.

🌐

Necesitas crearte una cuenta en my.zerotier.com para administrar la red. Solo el administrador necesita cuenta. Los jugadores solo instalan el cliente e ingresan el ID de red, sin registro.

// Instalar ZeroTier en Debian (servidor)

bash 
# Instalador oficial de ZeroTier
curl -s https://install.zerotier.com | sudo bash

# Unirse a tu red (reemplaza XXXXXXXXXX con tu Network ID)
sudo zerotier-cli join XXXXXXXXXX

# Ver estado de la conexión
sudo zerotier-cli status

# Ver la IP virtual asignada
sudo zerotier-cli listnetworks

// Asignar IP fija al servidor (desde el panel web)

  1. Ve a my.zerotier.com → entra a tu red
  2. En la sección Members verás el servidor listado
  3. Haz click en el dispositivo y en el campo Managed IPs escribe la IP que quieras, por ejemplo 10.147.17.1
  4. Esa IP nunca cambiará para ese dispositivo. Los jugadores siempre conectarán al servidor con esa dirección.

// En Windows (para los jugadores)

Descargan el instalador desde zerotier.com/download, lo instalan, abren ZeroTier en la bandeja del sistema, hacen click en Join Network y pegan el ID de la red. El administrador desde el panel web aprueba el dispositivo y listo.

💡

ZeroTier intenta siempre conexión P2P directa entre dispositivos. Solo usa servidores relay como fallback si la conexión directa falla. Esto minimiza la latencia.

06

RustDesk — Control remoto gráfico

RustDesk es el TeamViewer open source y gratuito. Permite ver y controlar el escritorio XFCE de la PC servidor como si estuvieras frente a ella. Funciona perfecto sobre la red de ZeroTier sin exponer nada al internet público.

ℹ️

RustDesk NO está en los repositorios oficiales de Debian. Hay que descargar el .deb directamente desde su GitHub.

// Instalar en Debian

bash 
# Descargar el .deb más reciente desde GitHub
# Verifica la versión actual en: github.com/rustdesk/rustdesk/releases
wget https://github.com/rustdesk/rustdesk/releases/latest/download/rustdesk-VERSION-x86_64.deb

# Instalar
sudo apt install -y ./rustdesk-VERSION-x86_64.deb

# Iniciar RustDesk (necesita estar corriendo para recibir conexiones)
rustdesk &

// Cómo usarlo

  1. En la PC servidor: abre RustDesk, te muestra un ID y una contraseña
  2. En tu PC: descarga RustDesk para Windows desde rustdesk.com
  3. Ingresa el ID de la PC servidor en el campo de conexión y la contraseña cuando la pida
  4. Ves el escritorio XFCE completo. Mouse, teclado, todo funciona.
💡

Para que RustDesk esté siempre disponible sin tener que abrirlo manualmente, configúralo como servicio del sistema con sudo systemctl enable rustdesk después de instalarlo.

07

SSH — Acceso rápido por terminal

SSH ya lo instalaste en el paso 03. Es la red de seguridad: si RustDesk falla por cualquier razón, siempre puedes entrar por SSH y arreglarlo desde la terminal.

// Configurar y activar SSH en la PC servidor

bash 
# Activar e iniciar el servicio SSH
sudo systemctl enable ssh
sudo systemctl start ssh

# Verificar que está corriendo
sudo systemctl status ssh

# Ver la IP virtual de ZeroTier (la usarás para conectarte)
sudo zerotier-cli listnetworks

// Conectarte desde tu PC (Windows)

cmd / powershell 
# Windows 10/11 ya trae SSH cliente nativo
# Reemplaza con tu usuario de Debian y la IP de ZeroTier del servidor
ssh tuusuario@10.147.17.1

# La primera vez te pregunta si confías en el host, escribes "yes"
# Luego te pide la contraseña de tu usuario en Debian
⚠️

SSH funciona a través de ZeroTier sin necesidad de abrir puertos en el router. Todo el tráfico va por la red virtual cifrada de ZeroTier. No hay que tocar la configuración del router.

08

Servidor Hytale — Migración y arranque

Los archivos del servidor de Hytale son exactamente los mismos en Windows y Linux. Solo cambia el script de arranque: el .bat se reemplaza por un .sh equivalente.

// Estructura de archivos

estructura 
~/hytale-server/
├── HytaleServer.jar       # el jar principal
├── Assets.zip             # assets del servidor
├── start.sh               # script de arranque (reemplaza el .bat)
├── server.properties      # configuración del server
└── worlds/                # mundos guardados

// De .bat a .sh — conversión de tu script

Tu .bat original en Windows:

windows · start.bat (original) 
@echo off
title El_Papu_Server
java -jar HytaleServer.jar --assets ..\Assets.zip --accept-early-plugins --auth-mode offline
pause

Equivalente en Linux (start.sh):

linux · start.sh 
#!/bin/bash
# Servidor Hytale
# -Xms: RAM mínima al arrancar | -Xmx: RAM máxima permitida

java -Xms4G -Xmx7G \
  -jar HytaleServer.jar \
  --assets ../Assets.zip \
  --accept-early-plugins \
  --auth-mode offline
ℹ️

En Linux las rutas usan / en vez de \. El @echo off, title y pause del .bat no tienen equivalente necesario en Linux, simplemente se omiten. El título de la sesión lo gestiona screen.

bash 
# Darle permisos de ejecución (solo una vez)
chmod +x start.sh

// Arrancar el servidor con Screen

Screen permite que el servidor corra en segundo plano y no muera cuando cierras la sesión SSH o la terminal. Es como minimizar sin cerrar.

bash 
# Iniciar el servidor dentro de una sesión screen llamada "hytale"
screen -S hytale ./start.sh

# Salir de screen SIN matar el servidor
Ctrl+A  →  D

# Volver a ver la consola del servidor
screen -r hytale

# Ver todas las sesiones screen activas
screen -ls

# Detener el servidor (dentro de screen)
Ctrl+C
💡

Flujo típico de administración remota: conectas por SSH → screen -r hytale para ver la consola en vivo → revisas que todo esté bien → Ctrl+A D para salir de screen sin afectar el servidor → cierras SSH. El servidor sigue corriendo sin interrupciones.

Hytale Server ≈ 7GB
Debian base ~300MB XFCE ~300MB (solo si está abierto) ZeroTier ~100MB Hytale disponible ~7GB
ℹ️

Ajusta los valores -Xms y -Xmx en el start.sh según la RAM real de tu PC. Regla general: deja al menos 700MB–1GB libres para el sistema operativo y reserva el resto para el servidor.

// Arranque automático del servidor con systemd

Para que el servidor de Hytale arranque solo cuando la PC enciende, sin que nadie tenga que hacer nada:

bash 
# Crear el archivo de servicio systemd
sudo nano /etc/systemd/system/hytale.service
hytale.service 
[Unit]
Description=Servidor Hytale
After=network.target zerotierone.service

[Service]
Type=forking
User=tuusuario
WorkingDirectory=/home/tuusuario/hytale-server
ExecStart=/usr/bin/screen -dmS hytale ./start.sh
ExecStop=/usr/bin/screen -S hytale -X quit
Restart=on-failure

[Install]
WantedBy=multi-user.target
bash 
# Activar el servicio para que arranque con el sistema
sudo systemctl daemon-reload
sudo systemctl enable hytale
sudo systemctl start hytale

# Ver estado
sudo systemctl status hytale
🎯

Con esto la PC arranca, ZeroTier se conecta automáticamente, el servidor de Hytale arranca solo, y los jugadores pueden conectarse sin que nadie en la ubicación del servidor tenga que hacer absolutamente nada. La PC es completamente autónoma.
